-----------------------------------
A. /announcements/summary 頁面需求
-----------------------------------

這頁是 announcement 總覽首頁，請做成高質感 SaaS 風格 dashboard。

區塊 1：KPI Cards（第一排）
- 今日總訊息數 totalMessagesToday
- 今日分析數 analyzedMessagesToday
- 待審核數 pendingReviewCount
- 已核准數 approvedCount
- 已退回數 rejectedCount

區塊 2：7 天趨勢
資料來源：
- GET /api/announcement-reports/weekly

至少做這些圖表：
1. 每日抓取量 / analyzedMessages / candidatesCreated 折線圖
2. approved / rejected 柱狀圖
3. 高信心公告數 highConfidenceCount 顯示

區塊 3：類型分布
- rule
- notice
- campaign
- discount
- script
- ignore（若後端 summary 有回傳才顯示，若沒有就隱藏）
可用 donut chart 或 horizontal bar chart

區塊 4：館別分布
- 以 facility 為單位顯示 count
- 排序由高到低
- 顯示前幾名即可，剩餘可折疊或滾動

區塊 5：系統狀態提示
- 若 summary API 成功但 weekly API 失敗，要顯示 partial success 狀態
- 若 API 為空，顯示「目前尚無公告分析資料」
- 若失敗，顯示 retry 按鈕

單筆 detail drawer / modal：

顯示：

- summary
- recommendedAction
